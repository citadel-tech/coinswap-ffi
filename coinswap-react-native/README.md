# Coinswap React Native

React Native JSI bindings for the Coinswap Bitcoin privacy protocol

</div>

## Overview

`coinswap-react-native` exposes the shared `ffi-commons` taker API to React Native apps through a JSI TurboModule generated by [uniffi-bindgen-react-native](https://github.com/jhugman/uniffi-bindgen-react-native).

The Rust library is compiled to native code for each platform target. The JSI layer gives JavaScript direct access to the native Rust objects with no serialisation overhead — no Kotlin/Swift bridge, no JNA, no ObjC reflection.

## Supported Targets

| Platform | Target | Use |
| --- | --- | --- |
| Android | `arm64-v8a` | Release — physical devices |
| Android | `armeabi-v7a` | Release — 32-bit legacy devices |
| Android | `x86_64` | Development — emulator |
| iOS | `aarch64-apple-ios` | Release — physical devices |
| iOS | `aarch64-apple-ios-sim` | Development — Apple Silicon simulator |
| iOS | `x86_64-apple-ios` | Development — Intel simulator |

## Building the Bindings

### Prerequisites

- Rust toolchain (`rustup`)
- Android NDK (`ANDROID_NDK_ROOT` or `ANDROID_HOME` set, or installed at the default macOS path `~/Library/Android/sdk/ndk/`)
- Xcode command-line tools (iOS builds)
- Node.js + npm

```bash
cd coinswap-react-native
npm install
```

### Using uniffi-bindgen-react-native (recommended)

These commands compile the Rust library and generate all native glue in one step.

```bash
# Android — compiles arm64-v8a, armeabi-v7a, x86_64 and generates Java/C++ JSI code
npm run ubrn:android

# iOS — compiles all iOS targets and generates ObjC++/C++ JSI code
npm run ubrn:ios

# Re-generate TypeScript + native glue only (no Rust recompile)
npm run ubrn:generate
```

Generated output (all gitignored, recreated by the commands above):

| Path | Contents |
| --- | --- |
| `src/generated/` | TypeScript types and `Taker` class |
| `src/NativeCoinswapReactNative.ts` | TurboModule codegen spec |
| `cpp/` | C++ JSI bridge |
| `android/CMakeLists.txt`, `android/cpp-adapter.cpp` | Android CMake build |
| `android/src/main/java/org/coinswap/` | Kotlin uniffi bindings + Java RN module |
| `android/src/main/jniLibs/` | Compiled Rust static libraries |
| `ios/CoinswapReactNative.{h,mm}` | ObjC++ RN module |
| `ios/coinswap_ffi.xcframework/` | Compiled Rust xcframework |

### Using the legacy build scripts

The shell scripts in `build-scripts/` compile a single Rust target each and copy the outputs manually. They are kept as a fallback and for CI targets not yet covered by ubrn.

```bash
# Android development build (x86_64 emulator)
bash ./build-scripts/development/build-dev-android-x86_64.sh

# Android release builds
bash ./build-scripts/release/build-release-android-arm64_v8a.sh
bash ./build-scripts/release/build-release-android-armeabi-v7a.sh

# iOS development build (host arch + iOS device + iOS simulator)
bash ./build-scripts/development/build-dev-ios.sh

# iOS release build (all targets, lipo universal simulator slice)
bash ./build-scripts/release/build-release-ios.sh
```

All scripts auto-detect the NDK host toolchain (`darwin-x86_64` on macOS, `linux-x86_64` on Linux). If `ANDROID_NDK_ROOT` is not set they fall back to `ANDROID_HOME/ndk/<latest>` or `~/Library/Android/sdk/ndk/<latest>`.

## Basic Usage

```typescript
import { AddressType, CoinswapTaker } from 'coinswap-react-native'

// Optional: set up file logging before creating a taker.
await CoinswapTaker.setupLogging(null, 'info')

// Create or load the taker wallet.
const taker = await CoinswapTaker.init({
  dataDir: null,                              // null uses the default taker directory
  walletFileName: 'my_wallet',               // wallet file to load or create
  rpcConfig: {
    url: 'localhost:18442',                  // Bitcoin Core RPC endpoint
    username: 'user',                        // Bitcoin Core RPC username
    password: 'password',                    // Bitcoin Core RPC password
    walletName: 'my_wallet',                 // Bitcoin Core wallet name
  },
  controlPort: 9051,                         // Tor control port
  torAuthPassword: 'coinswap',               // Tor control password
  zmqAddr: 'tcp://127.0.0.1:28332',          // Bitcoin Core ZMQ endpoint
  password: '',                              // optional wallet encryption password
})

// Sync wallet state and wait for the offer book.
await taker.syncOfferbookAndWait()
await taker.syncAndSave()

// Inspect balances and derive a receive address.
const balances = await taker.getBalances()
const address = await taker.getNextExternalAddress(AddressType.P2WPKH)

console.log('spendable:', balances.spendable, 'sats')
console.log('receive to:', address.address)

// Prepare the swap, then execute it with the returned swap id.
const swapId = await taker.prepareCoinswap({
  protocol: 'Legacy',                        // 'Legacy' | 'Taproot' | null for default
  sendAmount: 1_000_000n,                    // sats to swap (bigint — Rust u64)
  makerCount: 2,                             // number of maker hops
  txCount: 1,                                // number of funding transaction splits
  requiredConfirms: 1,                       // minimum funding confirmations
})

const report = await taker.startCoinswap(swapId)

console.log('swap id:', report.swapId)
console.log('status:', report.status)
console.log('outgoing:', report.outgoingAmount, 'sats')
console.log('fee:', report.feePaidOrEarned, 'sats')

// Release the taker when done.
await taker.dispose()
```

> **Note on numeric types**: amounts (`sendAmount`, `outgoingAmount`, etc.) are Rust `u64` and therefore `bigint` in TypeScript. Use the `n` suffix: `1_000_000n`.

## API Reference

### `CoinswapTaker.setupLogging`

```typescript
await CoinswapTaker.setupLogging(dataDir, level)
// dataDir  — directory for log files; null writes to stderr
// level    — 'trace' | 'debug' | 'info' | 'warn' | 'error'
```

### `CoinswapTaker.init`

```typescript
const taker = await CoinswapTaker.init({
  dataDir,          // string | null — taker data directory
  walletFileName,   // string | null — wallet file to load or create
  rpcConfig,        // RpcConfig | null — Bitcoin Core RPC settings
  controlPort,      // number | null — Tor control port
  torAuthPassword,  // string | null — Tor control password
  zmqAddr,          // string        — Bitcoin Core ZMQ endpoint (required)
  password,         // string | null — wallet encryption password
})
```

### `Taker` methods

```typescript
await taker.syncOfferbookAndWait()                         // block until the offer book is synced
await taker.syncAndSave()                                  // sync wallet state and persist it
const balances = await taker.getBalances()                 // read wallet balances
const addr = await taker.getNextExternalAddress(addrType)  // derive an external receive address
const swapId = await taker.prepareCoinswap(swapParams)     // prepare a swap and return its id
const report = await taker.startCoinswap(swapId)           // execute a prepared swap
await taker.dispose()                                      // release native resources
```

### `AddressType`

```typescript
AddressType.P2WPKH   // native SegWit (bech32)
AddressType.P2TR     // Taproot (bech32m)
```

### `SwapParams`

```typescript
{
  protocol?: string          // 'Legacy' | 'Taproot'; omit for backend default
  sendAmount: bigint         // total sats to swap
  makerCount: number         // number of maker hops
  txCount?: number           // number of funding transaction splits
  requiredConfirms?: number  // minimum funding confirmations
  preferredMakers?: string[] // optional maker addresses to prefer
}
```

### `Balances`

```typescript
balances.regular    // bigint — single-signature seed balance in sats
balances.swap       // bigint — swap-coin balance in sats
balances.contract   // bigint — live contract balance in sats
balances.fidelity   // bigint — fidelity bond balance in sats
balances.spendable  // bigint — regular + swap balance in sats
```

### `SwapReport`

```typescript
report.swapId           // string  — unique swap identifier
report.status           // string  — swap terminal state
report.outgoingAmount   // bigint  — sats sent by the taker
report.incomingAmount   // bigint  — sats received by the taker
report.feePaidOrEarned  // bigint  — negative when paid, positive when earned
report.makersCount      // number  — maker hop count used in the swap
report.makerAddresses   // string[] — maker addresses used in the route
report.totalMakerFees   // bigint  — aggregate maker fees in sats
report.miningFee        // bigint  — mining fees in sats
```

## Expo Plugin

The package ships an Expo config plugin at `app.plugin.js`. It configures the host app for AndroidX and React Native new architecture, and is the hook for future prebuilt binary distribution.

Add it to your `app.json` / `app.config.js`:

```json
{
  "expo": {
    "plugins": ["coinswap-react-native"]
  }
}
```

Or with options:

```js
{
  "expo": {
    "plugins": [
      ["coinswap-react-native", { "skipBinaryDownload": false }]
    ]
  }
}
```

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `skipBinaryDownload` | `boolean` | `false` | Skip the binary-artifact check during `expo prebuild`. Set to `true` if you are building the native library yourself. |

The plugin applies two modifications to the host project:

- **Android** — ensures `android.useAndroidX=true` and `newArchEnabled=true` are present in `gradle.properties`.
- **iOS** — no-op for now; iOS configuration is handled entirely by the podspec.

## Tests

Live end-to-end tests for both swap protocols are in `__tests__/`:

```bash
# Start the regtest environment first
cd ../ffi-commons
./ffi-docker-setup setup
./ffi-docker-setup start 4

# Run all live tests
cd ../coinswap-react-native
COINSWAP_LIVE_TESTS=1 npm run test:live
```

Tests require the shared regtest docker stack from `ffi-commons`.
